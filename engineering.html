Trust Contract

Fact, estimate, hypothetical

Recorded events are facts. Model outputs are estimates. Edited decisions are hypotheticals. The product should never visually or verbally mix those categories.

Overview

Fieldstate is being built as a football decision engine. It starts with real event data, explains pass/carry decisions, visualises actual chains of play, and adds modelled probability only when evidence gates are passed.

Current

Working demo layer

Real match moments, pass-chain replay, scenario editing, flat/perspective visuals, and evidence reports.

MVP

Gated probability layer

Use the approved event-only pass model in scenario output with provenance, calibration caveats, and safe copy.

Later

Tracking/neural layer

Add richer data, defender pressure, player movement, sequence models, and stronger counterfactual testing.

Delivery discipline

Engineering work is organised around reproducibility, controlled change and visible risk. The product roadmap lives in Feature Stages; this page explains how technical delivery is governed.

ControlWorking ruleInvestor relevance
ArchitectureKeep data, modelling, evidence and presentation boundaries explicit; document consequential decisions before scaling them.Reduces accidental coupling and makes later investment easier to diligence.
Source controlVersion code, public documentation, model releases and evidence artifacts; review changes before promotion.Creates traceability for what changed, why it changed and which claim it supports.
TestingProtect the critical demo, data parsing, scenario and evidence paths with automated checks and repeatable acceptance tests.Turns a founder-operated prototype into a handover-ready asset.
Security and data handlingSeparate public and private material, minimise secrets, validate inputs and document dataset permissions.Reduces disclosure, rights and operational risk before pilots or external access.
Technical debtMaintain a prioritised debt register; address debt when it threatens correctness, delivery speed, security or the next funded stage.Makes maintenance cost visible instead of hiding it inside feature estimates.
Roadmap decisionsAdvance models and infrastructure only when the current stage has a measurable outcome, evidence gate and explicit continuation decision.Protects capital from speculative complexity and premature scale.
