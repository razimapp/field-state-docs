# OpenFootball Public Football Data

- Source: https://github.com/openfootball/football.json
- Organisation: https://github.com/openfootball
- Date added: 2026-07-18
- Stage: Stage 3 - Data-Rich Decision Intelligence
- Capability category: public metadata / fixture context
- Licence noted by project: public domain / CC0-1.0 on GitHub repositories

## What It Can Help With

OpenFootball-style data is useful for public football context:

- competitions and leagues;
- fixtures and results;
- teams and match schedules;
- scorers in supported datasets;
- some broader football database exports.

## Fieldstate Use

Use it as a low-cost identity and context layer in the dream UI:

- player/team display names where available;
- fixture cards;
- match result context;
- season/competition navigation;
- public-data provenance examples.

## What It Does Not Solve

It should not be treated as a source for rich tactical intelligence:

- pass-option quality;
- pressure and lane obstruction;
- off-ball movement;
- possession value;
- probability of completion;
- neural sequence training;
- tracking/video-based analytics.

Those require event data, 360 freeze frames, tracking data, video-derived data, or licensed commercial datasets plus validation.

## Product Rule

Open/public data may label who and when. Fieldstate intelligence explains what decision happened, what options existed, and what the model can safely estimate.
