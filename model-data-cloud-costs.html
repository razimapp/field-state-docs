ML Models, development and cost breakdown

MVP can run lean because the first product surface uses event data, classical ML, and cached examples. The larger funding requirement appears when Fieldstate moves into licensed tracking data, neural sequence models, and continuous model operations.

Model ladder

Layer Purpose Compute profile Funding implication
Event-only baseline Pass completion, simple scenario probability, evidence gates. CPU-friendly. Train/evaluate locally or on small cloud CPU. Low cost. Mostly founder time and evaluation discipline.
Tree/boosting challengers Case-by-case cohorts, nonlinear interactions, reliability by pass type. CPU to modest GPU optional. Repeatable batch experiments. Low to moderate. Worth funding after MVP wiring.
Tracking-enriched models Defender pressure, receiver availability, lane quality, team shape. More data engineering; GPU not always required. Dataset licensing becomes the main cost and strategic dependency.
Neural sequence/spatial models Learn temporal context, player movement, off-ball structure, and richer counterfactuals. A100/H100-class GPU training and repeat experiments. Needs dedicated investor budget, validation, and model operations.
Production inference Serve probability, value, and scenario outputs reliably to users. CPU for classical models; small GPU only if neural inference is live. Can start lean, then scale with usage and model complexity.

Datasets to plan around

Open

StatsBomb Open Data

Useful for MVP evidence, replay examples, event features, and some 360 context. Not enough for full tracking-grade prediction.

Research

Metrica sample data

Useful to test support for event plus tracking data. Sample scale is limited, so it is not a commercial model corpus by itself.

Licensed

Provider tracking/event feeds

Needed for serious neural models and commercial accuracy claims. Pricing normally requires direct provider conversations.

Indicative cloud cost assumptions

Public GPU rates change and provider availability varies. Current public listings show why early classical ML is cheap, while serious neural training should be investor-funded.

Need Typical resource Public rate examples checked July 2026 Practical meaning
Classical model training CPU workstation or low-cost cloud CPU Hugging Face CPU endpoints list from about $0.03/hour to $0.54/hour for common CPU sizes. Not the budget problem. Evaluation quality matters more than raw compute.
Prototype GPU experiments A100/H100 single GPU Hugging Face lists A100 80GB at $2.50/hour and H100 80GB at $4.50/hour. Lambda lists H100 options around $3.29/hour to $4.29/hour. Good for short experiments, not continuous training without budget control.
Multi-GPU neural training 4 to 8 GPU nodes Hugging Face lists 8x A100 at $20/hour and 8x H100 at $36/hour. Lambda lists per-GPU H100/B200 rates by instance size. Costs rise quickly once repeated experiments, storage, and failed runs are included.
Production serving CPU service first; GPU only for neural inference Dedicated inference can start low, but model choice determines real cost. MVP should avoid locking the business into GPU-serving costs too early.

Sources checked: AWS EC2 On-Demand pricing, Google Cloud GPU pricing, Hugging Face pricing, and Lambda GPU cloud pricing. Licensed football data costs are excluded because providers usually quote privately.

Future advanced-model reference

This table is for investor planning only. MVP does not require this spend. These ranges describe what later funding could unlock once the product surface, data route, and evaluation gates are proven.

Future stage Dataset assumption Model type Indicative compute per run Why it matters
Feature-rich challenger Open event data plus engineered context features. Random forest, gradient boosting, calibrated ensembles. £50–£1k Improves case-by-case reliability, cohort testing, and investor confidence without heavy GPU spend.
Tracking-enriched model Licensed event plus tracking data for selected leagues or seasons. Spatial model using defender pressure, passing lanes, receiver availability, and team shape. £5k–£25k Moves Fieldstate closer to real football context rather than event-only estimation.
Neural sequence model Multi-season tracking with ball, player identities, possessions, and synchronized events. Transformer, temporal GNN, or hybrid sequence/spatial model. £25k–£100k+ Supports richer pass/carry prediction, counterfactual sensitivity, and tactical state understanding.
Research campaign Multiple competitions, repeated training runs, ablations, validation, and monitoring. Model family comparison with production candidate selection. £250k–£1.5m+ Creates a serious defensible model moat, but only after the commercial product route is validated.

These ranges are planning references, not quotes. They exclude licensed data fees, legal fees, staff time, failed experiments, long-term storage, and production support. Exact budgets should be re-priced when a provider, dataset scope, and model target are chosen.

Budget principle

MVP should prove the decision workflow with controlled evidence. Investors should fund neural training only after the product surface, dataset route, evaluation gates, and commercial use case are clear.
