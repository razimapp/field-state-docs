MVP Scope: what exactly gets delivered?

This document defines one bounded delivery sprint. It converts the existing prototype into a controlled, reviewable product asset with clear acceptance criteria and handover.

Later product stages, licensed data programmes and neural modelling are not part of this scope.

Definition of done

A credible end-to-end walkthrough

A non-technical reviewer can open a curated match moment, distinguish recorded evidence from estimates and hypotheticals, understand the result, and trace the supporting evidence.

Sprint outcome

At handover, Fieldstate should be demonstrable without improvised setup or unsupported claims.

Product

Controlled demo journey

A curated path from match context to recorded replay, one edited decision and an evidence explanation.

Evidence

Gated model output

Only an approved model output is surfaced, with provenance, calibration context and safe wording.

Delivery

Repeatable handover

Run instructions, tests, known limitations, source-controlled artifacts and a review checklist.

Acceptance criteria

Product acceptance

  • The selected demo match and moment load reliably.
  • The recorded action is never presented as hypothetical.
  • The edited branch is clearly labelled and visually distinct.
  • Copy is understandable without model or engineering knowledge.

Technical acceptance

  • Automated checks cover the critical demo and evidence paths.
  • Model artifacts and public claims are versioned together.
  • No secret or private material is required to run the public demo.
  • Known limitations and deferred debt are documented at handover.

Calendar duration and commercial terms should be agreed separately. The sequence defines delivery order, not a binding schedule.

Explicit exclusions

No full product platform

No multi-tenant production service, broad account system, live operations team or production-scale support commitment.

No new data-rights programme

No assumption of licensed tracking, video or private club data. The sprint uses approved current inputs.

No completed neural engine

No claim or delivery of neural sequence simulation, universal football prediction or tracking-grade counterfactuals.

Handover package

For investors and operators

Shareable demo route, capability boundary, evidence summary, limitations and recommended next decision.

For technical continuation

Repository state, setup and test instructions, architecture notes, model release references, security notes and prioritised technical debt.

Scope control

Any request that changes the acceptance criteria, introduces a new dataset, or expands the user workflow should be reviewed as a scope change rather than absorbed silently.
