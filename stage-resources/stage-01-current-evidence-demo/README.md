# Current Evidence Demo

## Purpose

Real event-data proof, pass-chain replay, controlled scenario edits, and evidence reporting.

## What Belongs Here

- Public links and references relevant to this stage.
- Product screenshots or visual references with source notes.
- Notes on user-facing features to build at this stage.
- Data/model requirements needed to support the features.
- Risks, caveats, and evidence gates before user-facing claims.

## Subfolders

| Folder | Use |
|---|---|
| `links/` | Markdown notes for external references and useful public resources. |
| `screenshots/` | Public-safe screenshots or exported images used for inspiration. |
| `notes/` | Stage-specific planning notes, feature ideas, and open questions. |
| `sketches/` | Rough diagrams, wireframes, and visual concepts. |

## Stage Guardrails

- Inspiration is not a capability claim.
- Label anything future-facing as future funded work.
- Do not store copied proprietary code, paid data, private legal strategy, or confidential investor comments.
- Convert inspiration into Fieldstate-specific requirements before implementation.

## Suggested Reference Note Format

```md
# Reference Title

- Source:
- Date added:
- Why it matters:
- Fieldstate inspiration:
- What we should avoid copying:
- Capability category:
- Data/model requirement:
- Follow-up action:
```
