# Fieldstate vs Traditional Football Analysis Tools

## Purpose

This note explains the product distinction investors should understand: many football tools help users analyse what happened. Fieldstate is being built to help users ask what else could have happened, adjust the decision, and estimate how the next phase may change.

No competitor names are required. The framing is category-level and investor-safe.

## Core Difference

| Category | Traditional Analysis Platform | Fieldstate Direction |
|---|---|---|
| Main question | What happened? | What if the decision changed? |
| User action | Watch, tag, search, clip, compare | Edit a pass/carry decision and compare likely consequences |
| Data role | Describe events and video | Ground scenarios in event/360/tracking data |
| Model role | Often descriptive or dashboard-led | Estimate completion, value, risk, continuation, and outcome paths |
| Investor story | Useful workflow tool | Decision-intelligence and simulation platform |

## Investor Message

Fieldstate should not be sold as another video or dashboard tool. The stronger story is that it can become the layer that sits above analysis: once a club knows what happened, Fieldstate helps evaluate the better decision, the missed option, and the likely effect of changing it.

For an ex-club-owner investor, the commercial appeal is simple:

- coaches get clearer decision review;
- players get more actionable improvement feedback;
- recruitment teams can evaluate decision quality, not only outcomes;
- media can explain pivotal choices more clearly;
- clubs can build proprietary football intelligence around repeatable decision moments.

## Stage 1 - Pass Decision MVP

### User Capability

- Select a real match and pass event.
- See the recorded pass and visible options from event/360 data where available.
- Edit the pass endpoint.
- Compare actual vs edited pass.
- Show pass-completion probability, value delta, turnover risk, and plausible continuation branches.

### Investor Appeal

This proves the core product loop without requiring full match simulation. The user sees the difference between replaying football and testing a decision.

### Boundary

Passes only. No corners, free kicks, penalties, or full neural simulation.

## Stage 2 - Pass Analysis Product

### User Capability

- Ranked pass-decision moments across a match.
- Player pass-decision profile.
- Team build-up decision review.
- Saved moments and exportable analyst reports.
- Evidence Centre showing model gates and calibration.

### Investor Appeal

This becomes pilot-ready for analysts and clubs because it creates repeatable workflows, not one-off demos.

### Boundary

Still primarily pass/carry decision intelligence. Set pieces remain future modules.

## Stage 3 - Wider Decision Intelligence

### User Capability

- Carry, turnover, and receiver-availability analysis.
- Momentum and possession-value timelines.
- Player/team decision patterns across matches.
- Similar-moment search: “show me when this player had a better option.”
- Analyst and coaching workflows around repeatable decision types.

### Investor Appeal

The product starts becoming a football decision layer rather than a pass-only tool. This is where club, academy, recruitment, and media use cases become more obvious.

### Boundary

Requires wider data, stronger evaluation, and careful model gating.

## Stage 4 - Set-Piece Modules

### User Capability

- Penalty probability and keeper/taker tendency modules.
- Corner setup analysis and delivery outcome estimates.
- Free-kick layout and shot/cross decision evaluation.
- Set-piece “what-if” adjustment: delivery zone, player run, marking shape, second-ball risk.

### Investor Appeal

Set pieces are commercially attractive because they are discrete, coachable, repeatable, and easy for clubs to understand. They can become premium modules.

### Boundary

Each set-piece type needs its own data, labels, model gates, and domain validation. They should not be bundled into MVP.

## Stage 5 - Full What-If Football Engine

### User Capability

- Multi-action scenario simulation.
- Player movement and team-shape adjustment.
- Tracking/video-informed off-ball context.
- Neural spatial/sequence models.
- Command Centre, Match Room, Counterfactual Lab, Scout Desk, and Evidence Centre working as one platform.

### Investor Appeal

This is the full dream: a football decision operating system for clubs, academies, recruitment, performance teams, and media storytelling.

### Boundary

Requires licensed tracking/video data, GPU compute, specialist ML, legal/data rights, and staged validation.

## Product Positioning Line

Traditional tools help clubs understand the match they played. Fieldstate helps them test the match they could have played.

## Stage Summary

| Stage | Product Door Open | What User Can Do | What Remains Future |
|---|---|---|---|
| 1 | Pass MVP | Edit and evaluate one pass decision | Set pieces, tracking, neural simulation |
| 2 | Pass product | Review ranked pass moments and player/team pass decisions | Full tactical engine |
| 3 | Decision intelligence | Add carries, turnovers, momentum, player decision profiles | Set-piece simulation and neural movement |
| 4 | Set-piece modules | Analyse penalties, corners, free kicks with dedicated models | Full multi-player continuous simulation |
| 5 | Dream engine | Simulate richer football scenarios with tracking/video/neural models | Production scale and continuous validation |

## Guardrail

The investor-facing story should be ambitious but staged. Fieldstate can show the full destination while making clear that MVP opens the pass-decision room first.
